  • Lecture 1 (ADT definition, Pseudocode, complexity definitions calculations)

    • Abstract data type definitions(pg 14)

      ADT:

      • A data type is a set of values (domain of the data type) and a set of operations on those values
      • the objects from the domain of the ADT are specified independently of their representation. The domain of an ADT describes what elements belong to this ADT.
      • the operations of the ADT are specified independently of their implementation
      • The set of all operations for an ADT is called its interface. The interface of an ADT contains the signature of the operations, together with their input data, results, preconditions and postconditions (but no detail regarding the implementation of the method).

      Containers

      Different containers are defined based on different properties:
      • do the elements need to be unique?
      • do the elements have positions assigned?
      • can any element be accessed or just some specific ones?
      • do we store simple elements or key - value pairs?
      A container should provide at least the following operations:
      • creating an empty container (constructor)
      • adding a new element to the container
      • removing an element from the container
      • returning the number of elements in the container
      • provide access to the elements from the container (usually using an iterator)

      Advantages to use ADTs

      • Abstraction is defined as the separation between the specification of an object (its domain and interface) and its implementation.
      • Encapsulation - abstraction provides a promise that any implementation of an ADT will belong to its domain and will respect its interface. And this is all that is needed to use an ADT.
      • Localization of change - any code that uses an ADT is still valid if the ADT changes (because no matter how it changes, it still has to respect the domain and interface)
      • Flexibility - an ADT can be implemented in different ways, but all these implementation have the same interface. Switching from one implementation to another can be done with minimal changes in the code.

    • Pseudocode cheatsheet(pg 34)

      Pseudocode is not object oriented, so instead of varible.fun(param1, param2), write fun(variable, param1, param2)
      • non standard instruction: @instruction
      • one line comment: //
      • reading data: read
      • printing data: print
      • assignment: ←
      • testing equality: = (single equation sign)
        • Arrays:
      • If we want to define a variable i of type Integer, we will write: i : Integer
      • If we want to define an array a, having elements of type T, we will write: a : T[]
      • Array indexing starts at 1
        • Pointers:
      • pointer: ↑
      • pointer definition: p: ↑ Integer - p is a variable whose value is the address of a memory location where an Integer value is stored.
      • The value from the address denoted by p is accessed using [p]
      • allocation: allocate(p), de-allocation: free(p)
      • We will use the special value NIL to denote an invalid address

    • Complexity definititions/calculations(pg 48)

      Each simple operation (+, -, *, /, =, if, function call) takes one time step/unit.

      T(n) = the number of steps

      We are not interested in the exact number of steps for a given algorithm, we are interested in its order of growth (i.e., how does the number of steps change if the value of n increases)

      We will consider only the leading term of the formula (for example n^2), because the other terms are relatively insignificant for large values of n.

      O notation

      The O-notation provides an asymptotic upper bound for a function: for all values of n (to the right of n0) the value of the function f (n) is on or below c · g(n)

      f(n) ∈ O(g(n)) if lim n→∞ f(n)/g(n) is 0 or a constant (but not ∞)

      Ω-notation

      The Ω-notation provides an asymptotic lower bound for a function: for all values of n (to the right of n0) the value of the function f (n) is on or above c · g(n)

      f(n) ∈ Ω(g(n)) if lim n→∞ f(n)/g(n) is ∞ or a nonzero constant.

      Θ-notation

      The Θ-notation provides an asymptotically tight bound for a function: for all values of n (to the right of n0) the value of the function f (n) is between c1 · g(n) and c2 · g(n)

      f(n) ∈ Θ(g(n)) if lim n→∞ f(n)/g(n) is a nonzero constant (and not ∞)
